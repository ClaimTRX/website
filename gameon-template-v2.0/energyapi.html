Rent Energy

Delegate energy on the TRON blockchain

Introduction

The user.js script is a Node.js client application designed to interact with the Energy API, a system for delegating energy on the TRON blockchain. This script allows users to place energy delegation orders by choosing between two payment methods: paying directly with TRX or using pre-deposited credits. The payment process is fully automated, requiring no manual intervention from the user—the script handles building, signing, and broadcasting the transaction using the configured private key wallet. The Energy API supports delegation of 65,000 energy units for 6.5 TRX or 135,000 energy units for 13 TRX, for a fixed duration of 5 minutes. This documentation provides a guide to the user.js script, including its purpose, setup, usage, and interaction with the Energy API.

Purpose

The user.js script enables users to:

  • Place an energy delegation order to a specified TRON wallet.
  • Choose between paying directly with TRX (automatically sent from the private key wallet) or using credits.
  • Delegate either 65,000 energy units for 6.5 TRX or 135,000 energy units for 13 TRX.
  • Monitor the status of the delegation order.

Setup and Configuration

Prerequisites

To use the user.js script, ensure the following are installed:

  • Node.js (v16 or higher): For running the script.
  • A TRON Wallet: With sufficient TRX for direct payments or credits deposited in the Energy API.

Script Code

Below is the complete user.js script, which you can copy and use:

const TronWeb = require('tronweb').TronWeb;

// === CONFIGURATION ===
const TRONGRID_API_KEY = "YOUR-TRONGRID-API-KEY"; // Replace with your TronGrid API key
const PRIVATE_KEY = "YOUR-PRIVATE-KEY"; // Replace with your private key
const API_KEY = "YOU-API-KEY"; // Replace with your API key
const USE_CREDITS = false; // Set to true to use credits, false to pay directly with TRX
const PAYMENT_WALLET = "TRUnBRHsGVYeFuBccYac5wyWYBAgcnLzmn"; // DO NOT CHANGE

// === ORDER SETUP ===
const order = {
    receiver: "RECEVING WALLET", // Replace with the receiving wallet address
    resource_type: "energy",
    amount: 65000,  // Options: 65000 or 135000
    duration: 5     // Duration in minutes (DO NOT CHANGE)
};

// === PRICE LOOKUP ===
const PRICE_TABLE = {
    65000: 6.5,
    135000: 13
};
const trxPrice = PRICE_TABLE[order.amount];
if (!trxPrice) {
    console.error("❌ Invalid order amount. Must be 65000 or 135000.");
    process.exit(1);
}
const priceInSun = trxPrice * 1e6;

// === TRON SETUP ===
const tronWeb = new TronWeb({
    fullHost: 'https://api.trongrid.io',
    headers: { 'TRON-PRO-API-KEY': TRONGRID_API_KEY }
});
tronWeb.setPrivateKey(PRIVATE_KEY);
const senderAddress = tronWeb.address.fromPrivateKey(PRIVATE_KEY);

// === API SETUP ===
const url = "http://api2.cftecosystem.com/energy";
const headers = {
    "Content-Type": "application/json",
    "Api-Key": API_KEY
};

// === MAIN FUNCTION ===
(async () => {
    console.log(`🔑 Sender: ${senderAddress}`);
    console.log(`💸 Paying ${trxPrice} TRX for ${order.amount} energy (Duration: ${order.duration} min)`);
    console.log(`➡️ To vendor: ${PAYMENT_WALLET}`);
    console.log(`🎯 Energy receiver: ${order.receiver}`);

    if (USE_CREDITS) {
        const res = await fetch(url + "/create-order", {
            method: "POST",
            headers,
            body: JSON.stringify({ order, use_credits: true })
        });
        const result = await res.json();
        console.log(result);
        return;
    }

    try {
        // === Build & Sign TX ===
        const tx = await tronWeb.transactionBuilder.sendTrx(
            PAYMENT_WALLET,
            priceInSun,
            senderAddress
        );
        tx.feeLimit = 1_000_000;

        const signedTx = await tronWeb.trx.sign(tx, PRIVATE_KEY);

        // === Broadcast TX ===
        const broadcastRes = await tronWeb.trx.sendRawTransaction(signedTx);
        console.log("📡 Broadcast response:", broadcastRes);

        if (!broadcastRes.result) {
            console.error("❌ Failed to broadcast transaction.");
            return;
        }

        console.log(`✅ TX broadcasted: ${signedTx.txID}`);

        // === Call /create-order with full signed TX ===
        const res = await fetch(url + "/create-order", {
            method: "POST",
            headers,
            body: JSON.stringify({
                order,
                use_credits: false,
                signed_tx: signedTx // Required by backend
            })
        });

        const text = await res.text();
        let data;
        try {
            data = JSON.parse(text);
        } catch (err) {
            console.error("❌ Failed to parse response:");
            console.log(text);
            return;
        }

        if (!res.ok || data.success === false) {
            console.error("❌ Order creation failed:", data);
            return;
        }

        console.log("✅ Order created:", data);

        // === Poll for Delegation Status ===
        const orderId = data.data.order_id;
        const statusUrl = `${url}/delegation-status?requestId=${orderId}`;
        let attempts = 0;

        const interval = setInterval(async () => {
            attempts++;
            try {
                const statusRes = await fetch(statusUrl, { method: 'GET', headers });
                const statusData = await statusRes.json();
                console.log(`ℹ️ Status [${attempts}]:`, statusData);

                if (statusData.status === "delegated") {
                    console.log(`✅ Energy delegated: TxID ${statusData.txId}`);
                    clearInterval(interval);
                } else if (statusData.status === "failed" || attempts >= 60) {
                    console.log(`❌ Delegation failed or timed out.`);
                    clearInterval(interval);
                }
            } catch (err) {
                console.error(`⚠️ Polling error: ${err.message}`);
                if (attempts >= 60) clearInterval(interval);
            }
        }, 2000);

    } catch (err) {
        console.error("❌ Transaction error:", err.message);
    }
})();

Setup Instructions

  1. Save the Script:
    • Copy the script above into a file named user.js.
  2. Install Dependencies:
    • Ensure Node.js is installed.
    • Install the required package: 
      npm install tronweb
    • Note: The script uses the native fetch API, available in Node.js v18 and higher. If using an older version, install node-fetch and modify the script to import it.
  3. Configure the Script:
    • Replace YOUR-TRONGRID-API-KEY with your TronGrid API key.
    • Replace YOUR-PRIVATE-KEY with your TRON wallet private key. This wallet will automatically send the payment for direct TRX orders.
    • Replace YOU-API-KEY with your Energy API key (e.g., 77b9c4e8a2f1d6b3e9a0c8f7d4e2b1c9a3d5e6f7b8c9a0b1c2d3e4f5a6b7c8d9).
    • Replace RECEVING WALLET with the TRON wallet address that will receive the delegated energy.
  4. Run the Script:
    • Execute the script using Node.js: 
      node user.js

Usage

The user.js script allows users to delegate energy by interacting with the Energy API. Users can choose to pay either with credits or directly with TRX, and the payment is automatically handled by the script using the configured private key wallet.

Payment Options

Users can choose between two payment methods when placing an order:

  • Direct TRX Payment:
    • The script automatically sends TRX from the wallet associated with the configured private key to the Energy API's payment address (TRUnBRHsGVYeFuBccYac5wyWYBAgcnLzmn).
    • The transaction is built, signed, and broadcasted without any user intervention.
    • Set USE_CREDITS = false in the script to use this method.
  • Credit Payment:
    • The user pays using credits previously deposited into the project's credit wallet (e.g., TQNJCwrnep6D11tMYoMceFVjn7MKRx1G95 for apitest).
    • Credits must be deposited via the Energy API's /deposit-credits endpoint (see the Energy API documentation for details).
    • Set USE_CREDITS = true in the script to use this method.

Pricing Structure

The Energy API supports the following pricing for energy delegation (fixed duration of 5 minutes):

  • 65,000 Energy Units: 6.5 TRX
  • 135,000 Energy Units: 13 TRX

Customizing the Script

To place an order, modify the following variables in user.js:

  • TRONGRID_API_KEY: Your TronGrid API key.
  • PRIVATE_KEY: Your TRON wallet private key (used to automatically send payments).
  • API_KEY: Your Energy API key.
  • order.receiver: The TRON wallet address to receive the delegated energy.
  • order.amount: The amount of energy to delegate (65000 or 135000).
  • USE_CREDITS: Set to true for credit payment, false for direct TRX payment.

Example Customization:

const TRONGRID_API_KEY = "your-trongrid-api-key";
const PRIVATE_KEY = "your-private-key";
const API_KEY = "77b9c4e8a2f1d6b3e9a0c8f7d4e2b1c9a3d5e6f7b8c9a0b1c2d3e4f5a6b7c8d9";
const USE_CREDITS = false;
const order = {
    receiver: "THNiVH2i5gqgTXR3PMYFaMKdygiXrzJPrk",
    resource_type: "energy",
    amount: 65000,
    duration: 5
};

Execution Flow

The script performs the following steps:

  1. Validate Order Amount: Ensures the order.amount is either 65,000 or 135,000.
  2. Process Payment:
    • If USE_CREDITS = false: Automatically builds, signs, and broadcasts a TRX payment to TRUnBRHsGVYeFuBccYac5wyWYBAgcnLzmn using the private key wallet.
    • If USE_CREDITS = true: Submits the order directly to /create-order, deducting from the credit balance.
  3. Create Order: Submits the order to the Energy API's /create-order endpoint.
  4. Monitor Status: Polls the /delegation-status endpoint every 2 seconds until the order is completed or fails.
